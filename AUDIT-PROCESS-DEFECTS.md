# full-audit v1.11 — Audit-Process Defects (meta-audit)

> Meta-auditor deliverable. Audits the AUDIT (full-audit v1.11, L3), NOT the target project.
> Target = offline C# Phoenix Point co-op mod (net472, Harmony single-DLL, Steam P2P, no web/DB/container).
> Evidence = the 14 agent JSONs in `audit/raw/*.json` (all "META:"-prefixed limitations harvested) + the spec files
> at `E:\DEV\Fullaudit\full-audit\` and workflow docs at `E:\DEV\Fullaudit\`.

## Summary — verdict

**Fit score: 42 / 100.**

full-audit v1.11's *generic recall-first engine* (adversarial-hunt, impact/logic/history reviewers, `suspected_unconfirmed` demotion, `coverage` block, and injection of the project `CLAUDE.md` sync-canon as `project_rules`) fit the project well and is the ONLY reason the six high-value bugs were found — every one of them (SEC-001/002/003, LOG-001, IMP-001/002) is `detection_method: manual|adversarial` and traces to domain reasoning or the injected canon, NOT to a spec checklist item. The *concrete spec content* fit poorly: ~70% of the universal.md/csharp.md checklist surface is web-service/deployable-oriented and inapplicable, ~80% of the pinned tooling was uninstallable or irrelevant, the entire reproduction phase was inert (nothing is runnable without the full game + Steam), and the severity/reproduction-override rules misfired on an offline mod — the framework had to be worked around, not followed.

**Claim verification (per the brief):** CONFIRMED. The high-value findings came from improvised checks. Cross-checking `detection_method` and cited origin in the JSONs: SEC-001 (manual, CWE-770 alloc-bomb), SEC-002/SEC-003 (manual/adversarial, P2P identity spoof), LOG-001 (manual, batch-abort desync), IMP-001/IMP-002 (manual, stuck-session). None cite a spec checklist section as origin. By contrast the spec-/tool-driven checks (Wave-1 CLI + the web-security sections) produced ONLY low-value output: CLIC-001..004 (UPnP SSRF, LOW), CLIC-007 (454 format violations), CLIC-005/006 (CI tag pinning), CLIU-001..004 (hardcoded SteamID/paths, stale branches), WEB-001..005 (dep-version gaps, none actionable). The checklists that fired caught nothing that mattered; the recall-first reviewers that improvised caught everything that did.

Defect counts by class: **(1) Spec Misfit — 9 · (2) Missing Coverage — 6 · (3) Tooling Gaps — 6 · (4) Workflow/Severity — 5 · (5) Process Contradictions — 3.**

## Class 2 — MISSING COVERAGE (risks that matter for a P2P co-op mod, with NO spec checklist item)

The biggest gap. These are the classes that produced every real bug — and the spec has no item for any of them. They were caught only because the recall-first reviewers improvised and because the project `CLAUDE.md` sync-canon was injected as `project_rules`. Verified against the spec: none of universal.md / csharp.md contains a checklist item for any row below.

| Defect (missing check) | Spec location (absent) | What happened (evidence) | Fix |
|---|---|---|---|
| Co-op state-sync desync / one-writer-per-field / dirty-mark-bypass / drift-poll coverage | NONE in universal.md or csharp.md | The entire high-value sync class was caught only via injected CLAUDE.md canon + improvisation: HIST-S01 (dirty-mark bypass, only 5 channels have a drift-poll backstop), ADV-001 (client suppresses all IHostOnlyApply outcomes; non-channelled event outcomes desync), CONV-001 (no cross-surface impact-frame barrier), CONV-S01 (one-writer-per-field not met for pos/AP-WP/health). | Add a "distributed/replicated-state correctness" L3 section: one-writer-per-field, authoritative-vs-mirror divergence, out-of-band mutation without a change signal, reconvergence/backstop coverage per channel. |
| P2P trust boundary where the PEER is the attacker (self-asserted id vs authenticated transport sender) | universal.md **"Level 3: IDOR / Access Control"** exists but is HTTP-endpoint framed ("every endpoint validates user access"); no peer-identity item | SEC-002 (LEAVE trusts payload steamId → any peer evicts any peer), SEC-003 (spoofed PlayerGuid in JOIN → identity takeover), SEC-005 (per-soldier ownership vacuous under blanket FullCommander). All adversarial/manual; the "authenticated transport sender vs self-asserted payload field" pattern is nowhere in the spec. | Add a "peer/message trust-boundary" security section: for every inbound message field used for authz, assert it is bound to the authenticated transport identity, not self-asserted. Generalize IDOR/BOLA off HTTP endpoints to "any authenticated channel." |
| Reflection / Harmony fragility across game or DLL updates | NONE (closest: universal.md **"Sharp Edges & Footgun Design"** #6 stringly-typed — too generic) | HIST-001 (1306 AccessTools sites, ~19 startup-guarded; a game rename → silent per-site desync), ADV-002 (faction-flipper exclusion is a 3-string exact-name match, DLC-fragile), LOG-002 (registry key with no value-equality → latent double-mint). | Stuck-state / no-recovery in the session/transfer state machine (single-threaded + coroutine) | universal.md **"Level 3: Business Logic & Domain Correctness"** names "no stuck states" but only generically; no P2P-lifecycle item | IMP-001 (phase-1 straggler kick strands client, no recovery), IMP-002 (null-buffer heartbeat pin), IMP-005 (heartbeat-timeout suspension disables host-loss detection), ADV-S01 (blocking-prompt gate soft-locks client). | Partial-apply / batch-abort correctness in a mirror stream | NONE | LOG-001 (HIGH): one throwing actor aborts the whole batch suffix AND skips the seq Mark → durable silent per-actor desync; sibling appliers use the correct per-record try/catch + Mark-after pattern. Pure variant-analysis improvisation. | Add a "streamed-batch apply" check: per-record isolation (one poison record must not drop the rest) + sequence/cursor must still advance so the stream cannot wedge. |
| Alloc-bomb via unbounded length prefix in hand-rolled binary codecs | universal.md **"Level 3: Deserialization Safety"** has "Input size limits" but the section is about type-confusion RCE (BinaryFormatter/pickle/yaml), not length-prefix DoS in custom BinaryReader framing | SEC-001/004/006/007 (CWE-770): `new byte[wireLen]` / `ReadBytes(wireLen)` with no bound across the frame decoder, JOIN manifest, deploy chunk reassembler, save reassembler. Manual. The deserialization table's examples don't cue a reviewer to hand-rolled length prefixes. | Add an explicit "untrusted length prefix → allocation" row (bound every wire count/length against remaining stream before allocating). Applies to any binary protocol, web or not. |

## Priority fixes for the audit developer (ranked)

1. **Ship a stack profile for non-web / offline / game-mod / plugin projects.** One `has_http_surface|has_db|has_container|has_package_manager|is_runnable` capability vector, detected in Phase 0, that gates the web-service sections (Class 1: HTTP headers, CSRF, JWT, session, XSS, SQLi, EF, Blazor, container, SBOM, CI-deploy) and the NuGet-dependent tooling (Class 3). This single change removes ~70% of the wasted checklist surface and ~80% of the inert tooling in one move.

2. **Add the missing high-value check families (Class 2) — this is where every real bug lived.** Four new L3 sections: (a) replicated/authoritative state-sync correctness (one-writer-per-field, out-of-band mutation, reconvergence coverage), (b) authenticated-channel trust boundary (self-asserted field vs transport-authenticated identity — generalize IDOR off HTTP), (c) runtime-binding/reflection drift for modding stacks, (d) liveness/recovery for stateful lifecycles (bounded wait + recovery path + notify-on-eviction). Evidence they'd have caught real bugs: SEC-002/003, HIST-001, IMP-001/002, LOG-001, ADV-001.

3. **Fix the reproduction/confidence model for non-runnable, latent-defect codebases.** Add a `STATIC_CONFIRMED` verdict and decouple severity from reproduction-confidence, so a structurally-certain, runtime-latent HIGH (HIST-001, IMP-003, ADV-S02) isn't parked under the 90 floor. Repurpose `repro_command`→`verification_command`. (Classes 3 + 4.)

4. **Make severity rules debt-aware.** The injected-canon "violation = HIGH minimum" must exempt debt the project's own design doc sanctions as incremental convergence (CONV-001, CONV-S01 were wrongly pushed toward HIGH). And make the "0-findings/>30%-SKIP → re-review" alarm stack-aware so legitimately-N/A sections don't trip it. (Classes 4 + 5.)

5. **Bless trust-boundary severity calibration and stack-prune the git-hygiene/BLOCKER rules.** Resolve the Class-5 contradictions: LAN/operator-local/offline downgrades are calibration, not softening; "tool not installed" is only a BLOCKER when the tool is applicable to the detected stack; prune the hard-coded `node_modules/.env/venv` artifact list to the stack's real artifacts. (Class 5 + Class 3.)
